# Profitability Next Steps Workflow

Use this workflow when user asks for:
- "next steps"
- "how to make this app profitable"
- "what should we build now"

## Steps

### 1. Load Context
- Read:
  - `PROJECT_STATUS.md`
  - `README.md`
  - `.agent/skills/algo-trading/references/profitability-roadmap.md`

### 2. Determine Current Phase
- Place project in one roadmap phase (0-5).
- If uncertain, default to earliest unfinished phase.

### 3. Produce Top 3 Tasks
For each task include:
- file paths to modify
- expected behavior change
- measurable acceptance criteria

### 4. Add Risk Gate
- Define one hard "stop/no-go" gate for this iteration.

### 5. State "Not Yet"
- Explicitly list items to avoid until current gate is passed.

## Output Format
1. Current phase
2. This sprint (3 tasks)
3. Acceptance criteria
4. Not yet
